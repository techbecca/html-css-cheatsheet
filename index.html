CSS Selectors and Specificity Cheatsheet

Selectors

Name Example Description
Type p Selects all <p>tags
Class .class Selects all elements with class="class"
Id #id Selects all elements with id="id"
Chaining selectors h3.section-heading Rule sets apply only to elements that match all criteria.
Descendant selectors div p Match elements that are descended (regardless of nesting level) from another matched selector.
Grouping selectors h1, h2 Rule sets apply to all selectors separated by commas.

Specificity

When determining which rule set will be applied to an element, the rule set with the most specific selector statement is applied.

Rule Description
How to determine specificity The fewer elements that can be matched to a selector statement, the more specific the selector statement is.
Selector specificity order The id selector is considered the most specific, then class, then tag.
Descendants The more levels of descendants that are used in selectors, the more specific it is.
Chaining The more chaining that occurs, the more specific it is.
The !important keyword The !important rule is used on declarations to override any other declarations for a property and ignore selector specificity.
Warning: Using the !important keyword is considered bad practice and should be used with caution.
